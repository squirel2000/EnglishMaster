## Context

現有介面自專案建立以來完全未設計，僅有瀏覽器預設樣式（黑白 Arial、無留白、無視覺層級）。使用者是查詢英文單字／片語／整句的中文母語學習者，工具訴求是「像查字典一樣好用、耐看」。使用者要求參考 Cambridge Dictionary 的元件配置（查詢框、結果卡片、發音按鈕）與一個瀏覽器擴充功能的內容呈現方式，並明確指定深色主題。

## Goals / Non-Goals

**Goals:**
- 建立一套主題一致的深色視覺語言（色彩、字體、間距），避免落入常見的「深色 SaaS 面板」樣板（深藍黑背景＋螢光強調色）。
- 讓查詢結果的資訊層級清楚可辨：headword、音標、詞性、釋義、例句各自有獨立的視覺角色。
- 在不更動任何既有商業邏輯、API 合約，也不更動測試所依賴的 accessible name／文字內容的前提下完成重新設計。

**Non-Goals:**
- 不實作淺色主題切換（使用者明確指定深色主題）。
- 不新增生字本／單字收藏等持久化功能（參考網站雖有此功能，但目前專案沒有對應的儲存後端；此次僅做視覺與排版重新設計，收藏功能留待未來另立提案）。
- 不更動任何 API 路由、資料來源或查詢邏輯。

## Decisions

### 色彩：深墨綠＋黃銅金，而非常見的深藍黑＋螢光色
採用深墨綠背景（`#0e1712`）搭配黃銅金主色調（`#cc9c46`），呼應「燙金書封字典」的意象；錯誤／警示訊息使用獨立的暗紅色（`#d17862`），與強調色分離。
- 考慮過的替代方案：近黑＋單一螢光強調色（acid-green／vermilion）——目前深色介面最常見的樣板配色；深藍黑 SaaS 面板配色——同樣通用，且與「字典／學習工具」情境無關。
- 選用黃銅金並只保留單一強調色，避免強調色與語意色（錯誤／警示）混用。

### 字體：三種字體角色分工，而非單一字體
Source Serif 4（headword、網站標題）／IBM Plex Sans（一般介面文字，含繁體中文系統字體後備）／IBM Plex Mono（音標、詞性標籤）。
- 考慮過的替代方案：全站單一無襯線字體（如 Inter）——堪用但無法呈現「字典」文本應有的層級（headword、音標、說明文字在紙本字典中本就是不同排版角色）。
- 透過 `next/font/google` 於建置時自動下載並自我託管字型檔（`display: swap`），避免執行期額外請求與版面位移。
- 繁體中文沒有對應的 Latin 字型涵蓋，交由瀏覽器逐字元 fallback 至系統中文字體（`Noto Sans TC`／`PingFang TC`／`Microsoft JhengHei`），屬瀏覽器原生行為，無需額外程式邏輯。

### 版面：單欄「詞條卡片」，而非表格或多欄呈現
查詢結果統一收斂為一張卡片（`entry-card` / `sentence-card`），內部以分隔線區分 headword 區塊與釋義／例句區塊；編號釋義使用 CSS `counter()` 自訂樣式取代瀏覽器預設列表樣式，以便控制顏色與字體並維持設計一致性。

### 保留既有 DOM 語意與文字內容以維持測試相容
所有變更僅新增 CSS class 與外層包裝元素、並以 inline SVG 取代 emoji，未變更任何測試所依賴的 accessible name（發音按鈕的 `aria-label`、「查詢」／「改用整句翻譯」／「改查字典」按鈕文字、結果標題的 heading 名稱）。裝飾性 SVG 一律標記 `aria-hidden`，不影響 accessible name 計算。

## Risks / Trade-offs

- [風險] 新增的範例查詢建議按鈕可能被誤解為全新功能 → [緩解] 已在 `lookup-query` capability 下以 ADDED Requirement 記錄為既有查詢入口的延伸，行為與手動輸入送出完全相同，未引入新的資料流程。
- [風險] `next/font/google` 於建置時需要網路存取才能下載字型檔 → [緩解] 屬 Next.js 標準建置流程；若離線建置環境無法存取 Google Fonts 會導致建置失敗，屆時需改用 `next/font/local` 搭配本機字型檔作為後備方案。
- [風險] 深色主題為單一主題、無淺色模式 → [緩解] 使用者已明確指定深色主題為需求；未來若需淺色模式，可在既有 CSS 變數（token）架構上新增第二組色彩變數，不需重構元件結構。

## Migration Plan

純前端視覺變更，無資料庫或 API 合約變更，無需資料遷移。部署方式與既有流程相同（`next build` → 部署）；若需回滾，直接回退此分支的 commit 即可，不影響其他分支或已部署資料。
